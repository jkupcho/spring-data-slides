Why Unit Test?

  • Executable Documentation
  • Quicker Feedback Loops
  • Push Releases Faster
  • GitHub badges!!

Why Spring Boot?

  • Easily Mock Spring Beans
  • Test entire application without deploying
  • Opinionated about library choices * still have the ability to be flexible
  • Syntactic Sugar

General Tips

  • Unit Tests should execute quickly
  • Coverge is important, but don't go crazy
  • Tests should consistently pass if it passes once, it should always pass unless you broke something
  • Test functionality at a granular levl, service and contract at a higher level
  • If your test *can* fail, flag it as integration
